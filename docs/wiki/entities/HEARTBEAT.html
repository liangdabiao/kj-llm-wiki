HEARTBEAT.md

OpenClaw的心跳检查清单，实现定期自检和状态监控。

基本信息

  • 类型：配置文件
  • 位置：workspace/HEARTBEAT.md
  • 作用：定期检查系统状态，推送重要信息给用户

核心作用

HEARTBEAT 不是定时任务，而是智能助理的主动关怀机制：

  • 知道何时不该打扰用户（沉默智能）
  • 专注于"有什么需要关注"
  • 固定间隔（默认30分钟）+ 主动唤醒

检查清单

1. 紧急邮件检查

  • 扫描未读邮件
  • 仅推送今日截止的行动项
  • 无紧急邮件 → 跳过

2. 近期日程提醒

  • 读取日历文件
  • 今日日程全部列出，明日仅列重要事项
  • 即将开始（<15分钟）特别提醒

3. 临时任务队列

  • 识别今日截止事项
  • 有截止日期 → 列出+剩余时间
  • 无紧急任务 → 跳过

4. 用户反馈记录

  • 检查钉钉消息、直接对话、邮件回复
  • 识别关键词：建议/不好/希望/优化
  • 有反馈 → 立即记录+确认收到

推送规则

推送（有警报）

  • 🔴 紧急邮件（今日截止）
  • 🔴 即将开始的会议（<15分钟）
  • 🟡 新的重要任务（今日截止）
  • 📝 用户反馈需要记录

不推送（HEARTBEAT_OK）

  • 无新信息
  • 无紧急事项
  • 例行检查无异常

OpenClaw会自动过滤HEARTBEAT_OK，用户不会收到

HEARTBEAT vs Cron

维度 Heartbeat Cron
触发方式 固定间隔30m+主动唤醒 cron表达式/固定间隔
任务来源 HEARTBEAT.md文件 配置中的job定义
执行模式 总是运行在主会话 main或isolated会话
LLM调用 每次都调用（除非跳过） 仅在agentTurn时调用
使用场景 运维检查、状态监控 定时任务、批处理
