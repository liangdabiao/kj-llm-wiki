RAG

检索增强生成（Retrieval-Augmented Generation），从本地知识库检索相关信息后生成回答。

基本信息

  • 类型：技术架构
  • 核心流程：意图分析 → 知识检索 → 回答生成
  • 作用：让AI基于准确的知识库回答，而非训练数据瞎猜

三阶段架构

阶段①：意图分析

  • 使用轻量模型（如qwen-turbo）
  • 提取用户问题中的关键词
  • 判断问题类型

阶段②：知识检索

  • 在本地知识库中进行多字段匹配评分
  • 无需向量数据库即可实现精准检索（适合26份文件以内）
  • 超过数百条建议引入FAISS等向量检索

阶段③：回答生成

  • 使用高性能模型（如qwen-max/claude-sonnet）
  • 基于检索结果生成自然语言回答

技术栈选型

组件 推荐方案 说明
大模型底座 阿里云千问/Claude 中文理解能力强
知识库存储 本地JSON/Markdown 轻量、可控
向量检索 QMD/FAISS 混合检索提升召回率
消息通道 企业微信+微信服务号 双端覆盖

RAG配置示例

knowledge_base:
  path: "./knowledge/faq"
  indexing:
    enabled: true
    chunk_size: 500
    overlap: 50
  retrieval:
    method: "vector"
    top_k: 3
    threshold: 0.7

混合检索策略

  • 语义向量检索：理解问题意图
  • 关键词BM25检索：精确匹配术语
  • 两者结合提升召回率

常见问题

Q: AI回答不准确？

A: 增加检索数量（top_k: 5），降低阈值（threshold: 0.6），启用重排序（rerank: true）

Q: 响应慢？

A: 使用更快的向量数据库（qdrant），使用更快的模型（Haiku速度是Sonnet的3倍）
