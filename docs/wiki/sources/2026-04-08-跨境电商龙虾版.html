OpenClaw跨境电商专用文件体系与40天养成经验。

核心观点

三层架构（跨境版）

第一层：身份层（写一次，全龙虾团队都懂你）

  • SOUL.md — 跨境电商龙虾的人格核心
  • IDENTITY.md — 龙虾的名片
  • USER.md — 用户信息（细节决定复利）

第二层：操作层（让龙虾稳定干活）

  • AGENTS.md — 根级操作规则，所有子龙虾继承
  • HEARTBEAT.md — 自愈机制

第三层：知识层（真正的护城河）

  • MEMORY.md — 长期记忆
  • 每日日志 — 短期记录
  • shared-context/ — 共享上下文

40天前后对比

40天前：

  • Listing文案堆砌emoji和"爆款必买"
  • 选品报告淹没在无效数据里
  • 客服回复千篇一律，转化率低
  • 花在纠错和手动检查的时间比操作Seller Central还多

40天后：

  • 每天早上8:01自动推送7条高潜力选品报告
  • 用用户语气生成多语言Listing
  • 24h监控inbox，智能回复询盘并自动同步订单到飞书表格
  • 下午5:00推送当日数据周报+次日优化建议
  • 8个子智能体协同工作

核心配置示例

  • SOUL.md定义"严谨外贸人"风格，避免中式英语和夸张营销
  • USER.md记录用户偏好：短段落、有力句子、数据优先、禁止营销腔
  • AGENTS.md规定每次会话启动必读顺序

关键概念

  • SOUL.md — AI人格配置
  • AGENTS.md — 操作规则
  • MEMORY.md — 长期记忆
  • HEARTBEAT.md — 心跳检查

与其他素材的关联

  • 与 OpenClaw从新手到进阶 中的SOUL/USER配置互补
  • 与 OpenClaw构建自我迭代AI助手 中的记忆系统相关
