# 反爬虫

> 网站为防止自动化程序抓取数据而采取的技术手段，以及爬虫开发者对应的绕过策略。

## 基本信息

- **类型**：技术概念
- **核心矛盾**：网站保护数据 vs 爬虫获取数据
- **应对原则**：模拟真实用户行为

---

## 常见反爬手段与应对

| 反爬手段 | 应对方法 | 难度 |
|---------|---------|------|
| User-Agent 检测 | 随机切换 UA，使用 fake-useragent 库 | 低 |
| IP 限制/封禁 | 代理 IP 池、降低请求频率 | 中 |
| 验证码（图片/滑块） | OCR 识别、打码平台、agent-browser --headed | 中 |
| 登录限制 | Cookie/Session 管理、requests.session() | 中 |
| JS 动态加载 | Selenium / Playwright / agent-browser | 高 |
| AJAX 异步数据 | 分析网络请求，直接请求 API | 中 |
| 请求频率限制 | 随机延迟、多 IP 轮换 | 低 |
| 字体反爬 | 解密自定义字体文件 | 高 |
| WebGL/Canvas 指纹 | 使用真实浏览器环境 | 高 |

## 知乎反爬实测记录

> 来源：2026-04-09 知识入库实测

在尝试抓取知乎问题 https://www.zhihu.com/question/20899988 时，测试了多种工具：

| 工具 | 结果 | 原因 |
|------|------|------|
| agent-browser（headless） | 403 `code: 40362` | 后端直接返回限制 JSON，前端未渲染 |
| Playwright MCP | 浏览器实例关闭 | 连接失败 |
| webReader（直接） | 仅返回 URL 元数据 | 内容为空 |
| agent-browser（no-sandbox） | 滑块验证码 | headless 被检测，需人工验证 |
| webReader（回答详情页） | 成功 | 通过回答详情页 URL 绕过 |

**结论**：知乎反爬非常严格，直接访问问题页会被拦截，但通过具体的回答详情页 URL（`/question/xxx/answer/yyy`）可以成功获取内容。
