# AI Harness

> AI 智能体外围的软件框架系统，负责组装提示词、提供工具、管理文件、运行命令、缓存记忆等"脏活累活"，是大模型从"聊天玩具"变为"生产力工具"的关键层。

## 基本信息

- **类型**：概念/架构
- **词源**：马具中的"挽具"或"缰绳"，意为控制和引导
- **提出者**：Sebastian Raschka 博士（系统性总结）
- **代表实现**：Claude Code、OpenClaw Cursor、mini-coding-agent

---

## LLM / Agent / Harness 关系

| 概念 | 比喻 | 职责 |
|------|------|------|
| **LLM** | 发动机 | 文本生成、推理 |
| **Reasoning Model** | 涡轮增压发动机 | 思考链（Thinking） |
| **Agent** | 自动驾驶系统 | 循环决策（规划→执行→纠正→完成） |
| **Harness** | 底盘+方向盘+仪表盘 | 文件读写、命令执行、工具管理、记忆缓存 |

三者关系：LLM 提供算力，Agent 循环驱动问题解决，Harness 提供运行时基础设施。

## 六大核心组件

### 1. 实时上下文 (Live Context)
- 下达指令时自动扫描代码库：项目类型、目录结构、最新提交
- 生成工作区摘要，打包给大模型
- 对应实践：[[CLAUDE.md]]（人设/系统指令）、[[SOUL.md]]（人格配置）

### 2. 提示词缓存 (Prompt Caching)
- 干湿分离：稳定前缀缓存复用 + 动态状态每次重新计算
- 稳定前缀：人设、系统指令、工具列表、项目概况
- 动态状态：聊天记录、短期记忆、当前指令

### 3. 工具 (Tools)
- 预设工具箱：查看/读取/修改文件、执行终端命令
- 安全检查：工具合法性、参数格式、高危授权、工作区限制

### 4. 上下文管理 (Context Management)
- 解决 Context Bloat（上下文膨胀）
- 暴力裁剪：截断过长日志/文档
- 去重与摘要：合并重复读取、浓缩历史记录

### 5. 会话记忆 (Session Memory)
- 双轨结构：
  - **完整日志（Full Log）**：一字不落存硬盘，用于留底和重建提示词
  - **工作记忆（Working Memory）**：精简版，只保留当前任务关键情报
- 对应实践：[[Memory]]（长期记忆系统）

### 6. 子智能体 (Subagents)
- 主 Agent 动态召唤子智能体处理边缘问题
- 子智能体继承部分上下文，但权限严格限制（只读/沙箱）
- 防止无限套娃和核心代码被破坏
- 对应实践：[[多Agent协作]]、[[AgentToAgent]]

- 真正让大模型蜕变的，不是模型本身，而是外围的 Harness 系统
