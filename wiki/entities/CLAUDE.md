# CLAUDE.md

> AI助手的岗位说明书，定义角色、工作流程、输出标准、用户偏好。

## 基本信息

- **类型**：配置文件/概念
- **来源**：13个AI助手文章
- **作用**：把运营经验"喂"给AI，让它按标准执行

---

## 核心作用

CLAUDE.md 是每个AI助手的"岗位说明书"，包含：
- **角色定义**：这个助手负责什么
- **工作流程**：收到输入后按什么步骤处理
- **输出标准**：最终输出什么格式的结果
- **用户偏好**：站点US为主、利润率30%以上、偏好品类等
- **异常处理**：遇到问题怎么办

---

## 示例：选品分析的CLAUDE.md

```markdown
收到ASIN后，自动依次调用5个数据接口：
1. 产品详情
2. 销量趋势
3. 用户差评
4. 竞品关键词
5. 流量结构

数据拉完后按固定模板输出报告：
- 市场规模
- 竞争格局
- 差异化机会
- 切入建议
```

## 不同素材中的观点

### 2026-04-08 13个AI助手
- 每个助手都有自己的CLAUDE.md
- 里面写清楚了角色、工作流程、输出标准、偏好
- 把运营经验"喂"给AI，让它按标准执行

### 2026-04-09 Karpathy-AI知识库搭建全流程
- Karpathy 称 schema 为知识库质量的关键因素："schema 写得好不好，比你选什么工具重要十倍"
- Schema 内容超级简单：没有数据库、没有插件、没有任何第三方依赖，一个纯文本文件加几条清晰规则
- 推荐包含：用途定义、目录约定、整理规范（一话题一文件、双链关联、INDEX.md）、关注方向

### 2026-04-10 AI Harness 到底是个啥
- CLAUDE.md 是 Harness"实时上下文"组件的落地形式：定义 Agent 的人设、系统指令、工具列表
- Harness 下达指令时自动扫描项目生成工作区摘要，CLAUDE.md 就是这个摘要的静态版本
- 提示词缓存中的"稳定前缀"正是 CLAUDE.md 的内容（人设+系统指令+项目概况）
