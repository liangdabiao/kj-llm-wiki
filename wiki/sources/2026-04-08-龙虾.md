## 核心观点

### 核心配置文件说明

| 文件 | 职责 | 说明 |
|------|------|------|
| AGENTS.md | Agent职责声明 | 决定工具权限 |
| SOUL.md | 个性化提示词 | 注入system prompt |
| TOOLS.md | 工具白名单/黑名单 | 安全边界 |
| IDENTITY.md | 身份标识 | name/avatar，通道展示 |
| USER.md | 用户偏好 | 上下文先验 |
| HEARTBEAT.md | 定时任务配置 | 可选 |
| BOOTSTRAP.md | 首次onboarding引导 | 一次性消费 |
| MEMORY.md | 用户记忆文档 | RAG源 |

### 关键区别
- **workspace里的文件**：管的是"这个Agent平时怎么干活"
- **openclaw.json里的配置**：管的是"这个系统怎么把它跑起来"

### 多Agent配置
- 每个Agent有独立的workspace
- main是主Agent，其他是子Agent
- 通过agentToAgent配置通信白名单

### 备份与恢复
```bash
# 创建完整备份（包含配置、记忆、技能、会话等所有数据）
openclaw backup create --verify
```

### 远程养虾
- 利用Tailscale穿透，在公司操作家里私有服务器
