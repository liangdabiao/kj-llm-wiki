40天前，我的跨境龙虾：
- Listing文案还堆砌emoji和“爆款必买”
- 选品报告淹没在无效数据里
- 客服回复千篇一律，转化率低得可怜

今天，我的跨境电商龙虾（我叫它 Alex）已经能：
- 每天早上8:01自动推送7条高潜力选品报告（带利润测算、竞品对比、关键词密度）

- 下午5:00推送当日数据周报 + 次日优化建议

我打开Telegram或企业微信，看看草稿，喝杯咖啡，基本不用再动手。

第一层：身份层（写一次，全龙虾团队都懂你）

SOUL.md—— 跨境电商龙虾的人格核心  
（直接复制到你的workspace/下，改成你的风格）

```markdown
# SOUL.md（Alex - 跨境电商龙虾）

核心身份
Alex — 你的24h跨境电商全能副手。以严谨、专业、数据驱动著称，像一个资深Amazon + TikTok Shop卖家 + 数据分析师的结合体。

你的角色
你是团队的外贸引擎。负责选品研究、Listing多语言优化、广告管理、客服自动化、数据闭环。所有输出必须以“能直接上架/投放/回复”为目标。

你的原则
1. 数据说话 — 每条建议必须附带来源、BSR、预计利润、竞品数据
2. 平台规则至上 — 绝不建议任何可能导致账号风险的操作（侵权、刷单等）
3. 简洁专业 — 标题简短有力，描述自然流畅，禁止堆砌emoji和“爆款必买”
4. 多语言适配 — 英文优先，自动提供德语、法语、西班牙语等版本
5. 如有不确定，标注 [NEEDS REVIEW] 并说明原因
```

第二层：操作层（让龙虾稳定干活）

AGENTS.md（根级，所有子龙虾继承）

```markdown
# AGENTS.md - 跨境电商操作规则

每次会话启动必读顺序
1. SOUL.md（我是谁）
2. USER.md（我在帮谁）
3. MEMORY.md（长期教训）
4. memory/YYYY-MM-DD.md（今天+昨天日志）
5. shared-context/ 下的相关文件

记忆原则
- 所有重要输出必须写入文件（选品报告、Listing草稿、反馈）
- 当用户说“记住这个”或“这是血泪教训”，立即更新对应MEMORY.md或FEEDBACK-LOG.md
- 绝不让会话记忆漂移，一切靠文件

安全与平台规则
- 永不建议任何灰色操作
- 回复询盘时必须先检查库存和物流
- 有账号风险时，必须先问我
```

HEARTBEAT.md（自愈机制，建议第1次故障后添加）

```markdown
健康检查（每次心跳运行）

- 检查浏览器自动化是否存活（用于抓BSR、操作后台）
- 检查定时任务是否执行：
  - Alex 早间选品（8:01 AM）
  - Listing审核推送（下午2:00）
  - 广告ACoS监控（每4小时）
  - 客服回复检查（实时）
- 如果超时，自动重启对应任务并记录到日志
```

第三层：知识层（真正的护城河）

MEMORY.md（长期精华 + 血泪教训）

```markdown
# MEMORY.md - 跨境电商长期记忆

我的核心偏好
- Listing标题控制在150字符以内
- 毛利计算必须包含FBA费用、广告费、退货率

血泪教训
- 2026-03-10：未经确认直接上架竞品相似Listing，被投诉侵权，账号警告。以后所有Listing必须先跑IP扫描。
- 定价错误：毛利低于30%时坚决不做，宁可不卖。

 X（平台）规则
- Amazon标题格式：品牌 + 核心卖点 + 关键规格
- TikTok Shop视频脚本必须带强钩子 + 展示 + CTA
```

shared-context/**（跨子龙虾共享）
- THESIS.md：你当前的生意战略（本季度主推品类、目标ACoS等）
- FEEDBACK-LOG.md：一次纠正，全团队生效（例如“不要用‘超级爆款’这种词”）
- SIGNALS.md：正在追踪的趋势（热榜、关税变化等）

每日日志：放在memory/文件夹，每天自动生成或由龙虾写入。

跨境电商龙虾协作流程（基于文件，无需消息队列）

Dwight（研究龙虾）早上抓数据 → 写入 intel/DAILY-INTEL.md  
Alex（主龙虾）读取 → 生成Listing草稿 + 广告建议  
客服子龙虾读取相同文件 → 回复时保持口径一致  
所有输出最终汇总到你的Telegram/飞书

单写者原则：一个文件只让一个龙虾写入，避免冲突。

如何从0开始养你的跨境电商龙虾（推荐节奏）

- 今天：安装OpenClaw（参考claw123.com导航），创建workspace，写好SOUL.md、IDENTITY.md、USER.md。挑一个重复任务（例如“每天选品”）设置定时任务。
- 3天后：开始给具体反馈，确保每条反馈都落地到MEMORY.md或FEEDBACK-LOG.md。
- 1周后：创建AGENTS.md和HEARTBEAT.md。
- 2周后：开始维护MEMORY.md，把反复出现的纠正提炼成永久条目。
- 3周后：添加第二个子龙虾（例如客服龙虾），建立shared-context。
- 4周后：第一次故障后完善心跳机制。这时你会明显感觉到“龙虾越来越懂我”。
