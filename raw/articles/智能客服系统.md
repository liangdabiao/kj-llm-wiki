为什么需要 AI 客服？
想象一下：你经营着一家电商或 SaaS 公司，客户来自全球各地。有人在美国问退款政策，有人在欧洲咨询功能使用，还有人在午夜时分提交技术支持工单。传统客服需要三班倒，成本高昂且响应时间不稳定。

而 AI 客服可以：

7×24 小时在线，永不疲倦
秒级响应，目标延迟低于 3 秒
95%+ 准确率处理常见问题
智能升级复杂问题给人工
多渠道统一，一次配置覆盖所有平台
OpenClaw 让这一切变得触手可及，而且完全掌控你的数据。

第一步：创建客服知识库
AI 客服的核心是知识库——它知道什么决定了它能回答什么。

1. 初始化知识库目录

mkdir -p knowledge/faq
cd knowledge/faq

方式一：上传现有文档

cp ~/documents/product-manual.pdf .
cp ~/documents/faq.xlsx .

创建配置文件 memory/knowledge-config.yaml：

knowledge_base:
  path: "./knowledge/faq"
  indexing:

    chunk_size: 500
    overlap: 50
  retrieval:
    method: "vector"
    top_k: 3
    threshold: 0.7

第四步：实现工单自动化
当 AI 无法解决问题时，需要自动创建工单并升级给人工。

工单系统 Skill
创建 skills/ticket-system.js：

// 工单系统接口示例
async function createTicket(customerIssue, customerInfo, priority) {
  // 连接到你的工单系统 API
  const response = await fetch('https://your-ticket-system.com/api/tickets', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${process.env.TICKET_API_KEY}`
    },
    body: JSON.stringify({
      subject: customerIssue.subject,
      description: customerIssue.description,
      customer: {
        name: customerInfo.name,
        email: customerInfo.email,
        phone: customerInfo.phone
      },
      priority: priority,
      source: 'ai-agent',
      assigned_to: getOnCallAgent()  // 获取当前值班客服
    })
  });
  
  return await response.json();
}

async function escalateToHuman(conversationContext) {
  const ticket = await createTicket(
    {
      subject: `AI 无法处理: ${conversationContext.topic}`,
      description: conversationContext.transcript
    },
    conversationContext.customer,
    'high'
  );
  
  return {
    message: `已为您创建工单 #${ticket.id}，我们的专家会在 2 小时内联系您。`,
    ticketId: ticket.id
  };
}

module.exports = { createTicket, escalateToHuman };
升级逻辑配置
在 Agent 配置中添加升级规则：

escalation_rules:
  - condition: "sentiment == 'negative' && confidence > 0.8"

    message: "我理解您的困扰。让我立即为您转接人工客服。"
  
  - condition: "topic not in knowledge_base && retries > 2"
    action: "create_ticket"
    message: "这个问题需要专家介入，我已为您创建工单。"

流程：

客户: 我买的东西不想要了，可以退款吗？

  ↓
客户: 好的，谢谢
  ↓

案例 2：技术支持智能升级
场景：客户遇到复杂的登录问题

流程：

客户: 我登录不了，提示密码错误但我没改过密码
  ↓
AI Agent:
  1. 尝试知识库检索：密码重置流程
  2. 提供标准解决方案
  
客户: 我试了重置，但收不到验证码
  ↓
AI Agent:
  1. 检测到知识库未涵盖
  2. 尝试第二个方案：检查垃圾邮件
  3. 询问邮箱提供商
  
客户: 都试过了，还是不行，这很急！
  ↓
AI Agent:
  1. 情感分析：检测到"急" + 失败重试
  2. 触发升级规则
  3. 创建高优先级工单
  4. 回复：
     "抱歉给您带来困扰。这个问题需要技术专家介入，
      我已为您创建紧急工单 #12345。
      我们的工程师会在 30 分钟内联系您。"
  ↓
后台: 工单系统 → 分配给技术团队 → 客户收到邮件通知

故障排查
问题 1：AI 回答不准确
原因：

知识库信息过时或不完整
检索阈值设置不当
解决方案：

# 调整检索参数
retrieval:
  top_k: 5  # 增加检索数量
  threshold: 0.6  # 降低阈值，召回更多结果
  rerank: true  # 启用重排序
问题 2：升级太频繁
原因：

升级规则过于宽松
知识库覆盖不足
解决方案：

# 添加重试逻辑
escalation_rules:
  - condition: "not_in_kb && retries < 2"
    action: "ask_for_clarification"
    message: "抱歉，我需要更多信息。您能详细描述一下吗？"
  
  - condition: "not_in_kb && retries >= 2"
    action: "escalate_to_human"
问题 3：响应慢
原因：

向量检索未优化
模型调用超时
解决方案：

# 使用更快的向量数据库
openclaw config set vector_db "qdrant"

# 使用更快的模型（简单 FAQ）
model: "claude-haiku-4.5"  # Haiku 速度是 Sonnet 的 3 倍
