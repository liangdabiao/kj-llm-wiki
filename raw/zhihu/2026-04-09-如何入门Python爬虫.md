## 高赞回答 2：谢科

"入门"是良好的动机，但是可能作用缓慢。如果你手里或者脑子里有一个项目，那么实践起来你会被目标驱动，而不会像学习模块一样慢慢学习。

另外如果说知识体系里的每一个知识点是图里的点，依赖关系是边的话，那么这个图一定不是一个有向无环图。因为学习A的经验可以帮助你学习B。因此，你不需要学习怎么样"入门"，因为这样的"入门"点根本不存在！你需要学习的是怎么样做一个比较大的东西，在这个过程中，你会很快地学会需要学会的东西的。

需要学习：
1. 基本的爬虫工作原理
2. 基本的http抓取工具，scrapy
3. Bloom Filter
4. 如果需要大规模网页抓取，你需要学习分布式爬虫的概念。其实没那么玄乎，你只要学会怎样维护一个所有集群机器能够有效分享的分布式队列就好。最简单的实现是python-rq
5. rq和Scrapy的结合：scrapy-redis
6. 后续处理，网页析取(python-goose)，存储(Mongodb)

### 爬虫工作原理

想象你是一只蜘蛛，现在你被放到了互联"网"上。那么，你需要把所有的网页都看一遍。怎么办呢？没问题呀，你就随便从某个地方开始，比如说人民日报的首页，这个叫initial pages。

在人民日报的首页，你看到那个页面引向的各种链接。于是你很开心地从爬到了"国内新闻"那个页面。太好了，这样你就已经爬完了俩页面（首页和国内新闻）！

突然你发现，在国内新闻这个页面上，有一个链接链回"首页"。作为一只聪明的蜘蛛，你肯定知道你不用爬回去的吧，因为你已经看过了啊。所以，你需要用你的脑子，存下你已经看过的页面地址。

Python 实现（伪代码）：

```python
import Queue
initial_page = "http://www.renminribao.com"
url_queue = Queue.Queue()
seen = set()
seen.insert(initial_page)
url_queue.put(initial_page)
while(True):
    if url_queue.size() > 0:
        current_url = url_queue.get()
        store(current_url)
        for next_url in extract_urls(current_url):
            if next_url not in seen:
                seen.put(next_url)
                url_queue.put(next_url)
    else:
        break
```

### 效率与 Bloom Filter

如果直接运行上面的代码，爬下整个豆瓣需要一整年。问题出在判重的复杂度：N个网站，判重 O(logN)。

Bloom Filter 可以使用固定内存以 O(1) 效率判定 URL 是否已在 set 中。特点：
- URL 不在 set 中 → 100% 确定没看过（不会漏爬）
- URL 在 set 中 → 大概率已看过（小概率重复爬）

### 分布式爬虫

100 多台机器昼夜不停运行一个月爬完豆瓣。架构：
- 99 台 slave（小机器）+ 1 台 master（大机器）
- master 上运行 Redis + rq 作为分布式队列
- master 上运行 Bloom Filter 进行 URL 判重
- 各 slave 安装 scrapy，向 master 请求/提交 URL

已有现成工具：scrapy-redis

### 后续处理

1. 有效存储（数据库设计）
2. 有效判重（网页级别去重）
3. 有效信息抽取（地址、正文提取）
4. 及时更新（预测网页更新频率）

### 项目实践：知乎投票分析器

为了验证某个知乎回答是否存在刷赞行为，从一个从未学过爬虫的人开始学爬虫。

关键技术点：
- 使用 requests.session() 维持登录态
- 使用正则表达式提取 JSON 数据
- 发现每个 JSON 页面都包含下一页地址（next 字段），空字符串即为最后一页
- Windows Py2 编码问题解法：在 site-packages 下创建 sitecustomize.py，添加 `sys.setdefaultencoding("utf-8")`

### 学习建议

- 入门目的不能太复杂，不要一上来就搞模拟登录或 JS 动态页面
- 先从简单的开始：爬百度贴吧、爬煎蛋、爬糗事百科
- 有了源码后：正则表达式提取 → BeautifulSoup 提取 → JSON 解析
- 逐步积累模块化的代码：登录过程、POST 数据、解析数据、写入 Excel
- "编程是最容易获得的超能力"
